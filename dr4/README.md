📸 ДР4 – Сървър за качване на снимки
🎯 Какво прави този сървър?
Позволява качване на снимка към телефонния указател. Снимките се записват локално във файловата система.
Сървърът предоставя URL, който може да се ползва от ДР3 (сървъра за контакти), за да показва снимката на даден контакт.

🚀 Стартиране (за разработка)
1. Инсталирай зависимости
bash
npm install
2. Стартирай сървъра
bash
node server.js
След успешен старт ще видиш:

text
DR4 server running on port 5001
Сървърът слуша на http://localhost:5001.

📡 API крайни точки
POST /upload
Качва снимка на сървъра.

Параметри (multipart/form-data):

Поле	Тип	Описание
photo	file	Снимката (JPG, PNG, GIF, WebP)
Пример за заявка (cURL):

bash
curl.exe -F "photo=@C:/path/to/photo.jpg" http://localhost:5001/upload
Примерен отговор (200 OK):

json
{
  "message": "Photo uploaded successfully",
  "filename": "6b498faf-5874-4880-b52d-10a30ed69ec9.jpg",
  "url": "/photos/6b498faf-5874-4880-b52d-10a30ed69ec9.jpg"
}
Възможни грешки:

Код	Описание
400	Липсва файл, непозволен тип или файлът е празен
413	Файлът е по-голям от 5 MB (ако е включено ограничение)
GET /photos/:filename
Връща самата снимка.

Пример:

text
GET http://localhost:5001/photos/6b498faf-5874-4880-b52d-10a30ed69ec9.jpg
Отговор:
Съдържанието на снимката с правилния Content-Type.
Ако файлът не съществува – връща 404 Not Found.

🔗 Интеграция с ДР3 (сървъра за контакти)
ДР3 трябва да:

Добави текстово поле photo (или photoFilename) към структурата на контакта.

При създаване/редактиране на контакт със снимка:

Изпрати снимката към POST /upload на този сървър.

Вземе filename от отговора.

Запази контакта с това име.

При показване на контакт – генерира URL за снимката:

text
http://localhost:5001/photos/{filename}
Няма нужда ДР4 да знае нищо за контактите. Двата сървъра са напълно независими – комуникацията става чрез споделяне само на низ (име на файл).

text
dr4-server/

├── package.json       # Зависимости (express, multer, uuid)
├── uploads/           # Папка, в която се записват качените снимки (създава се автоматично)
└── README.md          # Тази документация
⚙️ Конфигурация (ако искаш да промениш)
В server.js можеш да редактираш:

Порт – променливата PORT (по подразбиране 5001)

Максимален размер на файл – limits: { fileSize: 5 * 1024 * 1024 } (5 MB)

Разрешени разширения – масивът allowed в fileFilter

🧪 Тестване
Стартирай сървъра (node server.js).

Използвай:

Postman / Thunder Client – POST заявка с form-data поле photo.

cURL (PowerShell ползвай curl.exe).

HTML форма (приложена по-долу за бърз тест).

HTML тестова страница (test.html):

html
<form action="http://localhost:5001/upload" method="post" enctype="multipart/form-data">
    <input type="file" name="photo" accept="image/*">
    <button type="submit">Качи</button>
</form>
📌 Забележки за прототипа
Снимките се записват с уникални имена (UUID), за да няма колизии.

В прототипа няма автентикация, resize, защита от злонамерени файлове – това ще се добави при финализиране на проекта.

За финална версия се предвижда запис в облачно хранилище (S3 и пр.) или база данни.
